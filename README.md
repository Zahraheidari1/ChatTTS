# ChatTTS — Conversational Text-to-Speech

ChatTTS is a neural text-to-speech (TTS) model built for dialogue and LLM assistant applications. It generates natural-sounding speech with fine-grained prosody control: laughter, pauses, and oral expressions.

- **Languages**: English (experimental) + Chinese + **Farsi / فارسی** (via Finglish pipeline — see [docs/fa/README.md](docs/fa/README.md))

- **Architecture**: GPT-based autoregressive model + DVAE codec + Vocos vocoder
- **Training data**: 40,000+ hours of speech
- **RTF**: ~0.3 on RTX 4090
- **Minimum VRAM**: 4 GB
- **License**: AGPLv3 (code) / CC BY-NC 4.0 (model weights — research use only)

## Prerequisites

- Python 3.10–3.12
- CUDA-capable GPU with 4 GB+ VRAM (CPU also works, but slow)
- [Anaconda / Miniconda](https://docs.conda.io/en/latest/miniconda.html)

## Installation

### 1. Create and activate the conda environment

```bash
conda create -n speech python=3.12 -y
conda activate speech
```

### 2. Install PyTorch (CUDA 12.8 example — adjust for your CUDA version)

```bash
pip install torch torchaudio torchvision --index-url https://download.pytorch.org/whl/cu128
```

For CPU-only:
```bash
pip install torch torchaudio torchvision
```

### 3. Install project dependencies

```bash
cd ChatTTS-main
pip install -r requirements.txt
pip install av          # PyAV — required for MP3/WAV export
```

### 4. (Optional) Install the package in editable mode

```bash
pip install -e .
```

## Model Download (where are the speech models?)

**You do not need to download models manually.** On first run, ChatTTS auto-downloads them.

There are three download modes. Use `--source` to select:

| Mode | What it does | Use when |
|---|---|---|
| `huggingface` | Downloads via HuggingFace Hub into `~/.cache/huggingface/` | First run, internet available |
| `local` | Downloads into `asset/` folder next to this README | You want a local copy |
| `custom` | Loads from a path you provide | You already have the files |

**Model files location after download:**

```
ChatTTS-main/
└── asset/                      ← created automatically with --source local
    ├── Vocos.safetensors
    ├── DVAE.safetensors
    ├── Embed.safetensors
    ├── Decoder.safetensors
    ├── gpt/
    │   ├── config.json
    │   └── model.safetensors
    └── tokenizer/
        ├── tokenizer.json
        ├── tokenizer_config.json
        └── special_tokens_map.json
```

Or HuggingFace cache:
```
~/.cache/huggingface/hub/models--2Noise--ChatTTS/
```

## Running the Code

All commands must be run from the `ChatTTS-main/` directory with the `speech` env active.

```bash
conda activate speech
cd ChatTTS-main
```

### Option 1 — Command Line (quickest)

```bash
# Download models from HuggingFace on first run, then generate speech
python -m examples.cmd.run --source huggingface "Hello, this is ChatTTS speaking."
```

Output is saved as `output_audio_0.mp3` in your current directory.

Multiple sentences:
```bash
python -m examples.cmd.run --source huggingface "First sentence." "Second sentence."
```

Use a local asset folder (downloads there on first run, reuses on next runs):
```bash
python -m examples.cmd.run --source local "Hello world."
```

Streaming mode:
```bash
python -m examples.cmd.run --source huggingface --stream "Hello world."
```

### Option 2 — Web UI (Gradio)

```bash
python examples/web/webui.py
```

Opens a browser UI at `http://localhost:7860` where you can type text, adjust voice parameters, and play audio in real time.

### Option 3 — Python API

```python
import ChatTTS
import soundfile as sf

chat = ChatTTS.Chat()
chat.load(source="huggingface")          # downloads models on first call

wavs = chat.infer(["Hello, how are you?"])
sf.write("output.wav", wavs[0], 24000)   # save as WAV
```

### Option 4 — REST API (OpenAI-compatible)

```bash
pip install fastapi uvicorn
python examples/api/openai_api.py
```

The server starts at `http://localhost:9966`. Compatible with OpenAI `/v1/audio/speech` clients.

### Option 5 — Jupyter Notebook

Open `examples/ipynb/example.ipynb` or the root `openai_api.ipynb` in Jupyter:
```bash
jupyter notebook
```

## Voice / Prosody Control

### Speaker embedding

```python
# Random voice
spk = chat.sample_random_speaker()

# Reuse a specific voice across sessions (save the string)
print(spk)  # copy and paste into future sessions

wavs = chat.infer(["Text here"], params_infer_code=ChatTTS.Chat.InferCodeParams(spk_emb=spk))
```

### Oral & laughter tokens (insert directly in text)

| Token | Effect |
|---|---|
| `[oral_0]` – `[oral_9]` | Oral expression level (0=least, 9=most) |
| `[laugh_0]` – `[laugh_2]` | Laughter intensity |
| `[break_0]` – `[break_7]` | Pause duration |
| `[uv_break]` | Short micro-pause |

Example:
```python
text = "Hello [uv_break] how are you [laugh_0] doing today [oral_3]?"
wavs = chat.infer([text])
```

## Troubleshooting

**"ModuleNotFoundError: No module named 'av'"**
```bash
pip install av
```

**"Models load failed" / asset folder missing**
- Run with `--source huggingface` to download from HuggingFace Hub automatically.
- Or run with `--source local` to download into the `asset/` folder.

**"CUDA out of memory"**
- Reduce batch size or use CPU: set `device=torch.device("cpu")` in `chat.load()`.

**Low audio quality / noise**
- This is intentional — the open-source model includes high-frequency noise as an anti-misuse measure.
- Sample multiple times; autoregressive models can be unstable on the first attempt.

**Text normalization warnings (nemo / WeTextProcessing)**
- These are Linux-only packages. On Windows, they are skipped automatically and text is passed through without advanced normalization. This is fine for most inputs.
